Introduction

This is the MSP430_5xx_6xx Peripheral Driver Library.    MSP430ware will allow you to develop your applications faster by providing all pertinent MSP430 related examples/documentation in one package and providing a peripheral driver library  for the MSP430 peripherals.  This peripheral library's intent is to allow application development at an API level instead at the device register level.   Allowing developers to concentrate on their applications instead of the nuances of the particular MSP430 device they are using.

    • Initial Release of MSP430ware

    The MSP430ware contains the following components.

    doc Device and MSP430 Peripheral Driver Library documentation
    driverlib MSP430 Peripheral Driver Library source code.  A pre-built version of the library and the vendor's project files may be found in the compiler vendor's subdirectory (i.e. CCS will contain the CCS build of the Driver library)
    examples/boards Board and User Experience Code
    examples/devices/?xx Device specific example code (non-MSP430 Peripheral Driver Library based examples)
    examples/driverlib MSP430 Peripheral Driver Library peripheral specific code examples
    examples/grace Future Grace Examples
    grlib Future MSP430 Graphics Library
    inc MSP430 Driver Library include files
    SimpliciTI Future location for TI's SimpliciTI RF communication stack
    TIRexCCS Resource Explorer files
    LicensesMSP430ware license text file
    usblib430 MSP430 USB Stack

    Installation and Usage

    Download and run MSP430ware-1.00.00.00-Setup.exe.  Follow the installation instructions.  By default, MSP430ware will be installed in c:/ti/msp430/MSP430ware_1_00_00_00.   Please see examples/driverlib/5xx_6xx/{peripheral} as a starting point for using MSP430ware.

    Upgrade and Compatibility Information

    Initial release

    This release uses the header files in the deprecated\IAR and deprecated\CCS folders instead of the default IAR and CCS header files.
    The hw_memmap.h takes care of this mapping and as long as the user includes this header file in their examples, the correct header file is mapped in.

    In future releases, when the default msp430 specific header files of CCS and IAR get updated, the deprecated folder will be removed from the MSP430ware package. The user examples will NOT be affected by any of these changes, since hw_memmap.h will re-direct to the correct files.

    To update MSP430ware via CCS 5.1 please follow these instructions.

    1.  Start CCS
    2.  Help->Install New Software...
    3.  In the "Work with:" drop down, select
        MSP430ware Updates - http://software-dl.ti.com/msp430/msp430_public_sw/mcu/msp430/MSP430ware/update/latest/
    4. Check MSP430ware (after pressing "+" next to Code Composer Studio)


    5.  Press Next and accept the license agreement
    6.  Press Finish
    7.  After the update downloads, CCS will ask for a restart.
    8.  Run through the installation and CCS will then come up with the new version of MSP430ware.

    Examples

    The examples may be built with either IAR or CCS.  To build the examples navigate to examples/driverlib/5xx_6xx/<peripheral> and double click the .eww project files for IAR or .project file for CCS. 

    Rebuilding the library

    The library may be rebuilt by navigating to driverlib/5xx_6xx/IAR or driverlib/5xx_6xx/CCS directory and double clicking on driverlib.eww of IAR or driverlib.prj for CCS.    

    The driverlib is by default built for the large data model. If your application requires a different data model, please navigate to the IAR or CCS project/folder in driverlib/5xx_6xx/ and change the data model in the project settings and rebuild the library from the IDE.
